- (decision) AD-1 Paradigm: monolith-first backend. One Gleam/BEAM app is the only runtime in v1. No microservices, no GraphQL, no Kubernetes. [ADOPTED]
- (decision) AD-2 Frontend is BARE React Native (CLI, native iOS/Android builds). Explicitly NOT Expo. [ADOPTED]
- (decision) AD-3 Backend: Gleam on BEAM is primary (HTTP + websockets + OTP). Rust is NIFs/hot-paths ONLY (LLM stream parse, geo math, image later). Keep NIF surface minimal. [ADOPTED]
- (decision) AD-4 State: Postgres = system of record (the 'adamas database'); Redis = cache/sessions/WS pub-sub fan-out; RabbitMQ = durable async (LLM jobs, notice fan-out, agent tasks). [ADOPTED]
- (decision) AD-5 RabbitMQ is wired only at the first genuine durable-queue need (LLM-agent pipeline). Redis pub/sub covers realtime day one. [ADOPTED]
- (decision) AD-6 LLM behind one provider abstraction with OpenRouter + Amazon Bedrock backends, selected at runtime. [ADOPTED]
- (decision) AD-7 Auth: login validates roll number + credential against adamas DB (Postgres). On match, school/department/year is known and gates dept-year chat room admission. No separate manual approval in v1. [ADOPTED]
- (decision) AD-8 Realtime (chat/notices/group) = websocket backed by Redis pub/sub fan-out. History persisted in Postgres. [ADOPTED]
- (decision) AD-9 Project state is the source of truth in openspec/ change dirs. GitHub Issues are a human-facing tracker mirror only, never read as state. [ADOPTED]
- (decision) AD-10 Deferred (NOT in v1): photobooth, AI map navigation, bus navigation, location punch-in/out, Kafka. [ADOPTED]
